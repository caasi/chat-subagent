- [ ] **Step 3: Add Question Operator examples section**

After the "Feedback Loop" examples section (line ~138), add:

```markdown
### Question Operator

`?` marks a step as producing Either, feeding into `|||` for branching:

\`\`\`
fetch(url: primary)? ||| fetch(url: mirror)
\`\`\`

`?` can appear upstream in a `>>>` chain that feeds into `|||`:

\`\`\`
loop(
  generate >>> verify >>> "all tests pass"?
  >>> (done ||| fix_and_retry)
)
\`\`\`

Only the "try" side (left operand of `|||`, or the upstream step producing Either) gets `?`. The fallback side does not.

The checker emits a warning (to stderr, exit code unaffected) if `?` appears without a matching `|||` in scope.
```

- [ ] **Step 4: Update Structural Rules section**

Add after the existing "The checker does NOT validate:" list:

```markdown
### Warnings

- `?` without matching `|||` in scope — the Either has no consumer
```

Also update the introductory line from "The checker validates **syntax structure** only:" to "The checker validates **syntax structure** only, and emits warnings:".

- [ ] **Step 5: Remove stale loop evaluation reference if present**
